# Full n8n + Claude Workflow Pack — public listing preview

This page is the public, checkout-disabled listing preview for the broader `n8n + Claude Workflow Pack` that the Free Lite workflow samples.

It is meant to answer “what would the paid pack include?” without collecting payment, requesting private data, or promising production outcomes. The current full pack is still a local draft package; this public page does not publish the paid ZIP, activate checkout, or create a payment flow.

## Who it is for

- Operators who already use n8n and want review-first Claude workflow templates instead of fully autonomous posting or mutating automations.
- Engineering, ops, or solo-consulting teams that need weekly GitHub summaries, PR risk checks, Slack/Notion-ready briefs, meeting action extraction, and safe audit/backlog helpers.
- Buyers who can evaluate with public repositories, sanitized payloads, or synthetic sample data before connecting private workspaces.

## Buyer self-check before inquiring

A good public inquiry can answer these without sensitive details:

1. Which path are you evaluating: Free Lite, Starter/Pro template pack, audit, or pilot?
2. Are you using self-hosted n8n, n8n Cloud, or still deciding?
3. Which outcome matters most: weekly status, PR review, Slack/Notion brief, meeting actions, KB/RAG gaps, audit backlog, or lead/scope triage?
4. Can you test first with a public repository or synthetic sample payload?
5. Do you need templates only, or separate implementation help later?

## Public-safety boundary

Public issues and screenshots must not contain tokens, credentials, private repository URLs, customer data, production logs, procurement/legal documents, checkout/payment details, KYC/tax information, bank, wallet, payout details, contracts, private workflow exports, or confidential screenshots.

No guaranteed ROI is claimed. The workflows are templates and draft/review helpers; buyers remain responsible for their own n8n instance, API credentials, workspace permissions, security review, legal review, and production approvals.

## What would need approval before a real paid transaction

A public listing preview like this can stay public-safe, but the following remain blocked until separately approved and scoped:

- activating checkout or payment capture;
- entering payout, wallet, bank, Stripe, tax, KYC, or contract details;
- paying platform fees, subscriptions, or ads;
- using private/client/workplace data;
- private outreach by DM, email, form submission, or sales application;
- making support, refund, SLA, legal, or guaranteed ROI promises.

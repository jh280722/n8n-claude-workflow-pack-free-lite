# Public share kit for Free Lite

Use this page when sharing the Free Lite workflow in a public issue, discussion, changelog, or community thread. It keeps outreach public-only, useful, and safe for readers who are not ready to share private workflow details.

## When it is appropriate to share

Share the repo only when the thread or venue is explicitly about one of these problems:

- summarizing public GitHub activity for a weekly team update;
- testing an n8n import with a small read-only workflow;
- comparing a public-repo snapshot before adding Slack, Notion, Discord, or Claude-style narrative steps;
- looking for safe examples of approval-first automation patterns.

Do not post generic promotion where the thread is unrelated, where self-promotion is disallowed, or where the only next step would be DM/email/form/private outreach.

## Public-safe short copy

```markdown
If you want a small n8n example to test the workflow style first, I keep a free read-only GitHub weekly snapshot workflow here:
https://github.com/jh280722/n8n-claude-workflow-pack-free-lite

It works with a public repo or sanitized sample data, outputs Markdown for review, and does not post to Slack/Notion or mutate GitHub by default. Please do not share tokens, private repository URLs, customer data, production logs, or credentials in public issues.
```

## Public-safe longer copy

```markdown
For teams evaluating n8n + Claude-style ops automation, this Free Lite repo provides a minimal read-only starting point: an importable n8n workflow that turns public GitHub activity into a weekly Markdown snapshot.

Useful if you want to check import/setup friction and output quality before connecting private workspaces, Slack, Notion, or Claude/API keys. The paid/full-pack page is only a checkout-disabled preview; there is no active checkout or payment capture in this repo.

Public issue safety: use public repos or sanitized examples only. Do not post tokens, credentials, private repository URLs, customer data, production logs, checkout/payment details, KYC, tax, bank, contracts, or guaranteed ROI requirements.
```

## Hard boundaries

Public sharing must not include or request:

- tokens, API keys, credentials, private repository URLs, customer data, production logs, private workflow exports, or confidential screenshots;
- checkout/payment activation, payment capture, payout/wallet/bank/Stripe/tax/KYC/contract details, invoices, or platform-fee steps;
- DM, email, forms, private outreach, paid ads, subscriptions, or spend;
- legal, SLA, support/refund, or no guaranteed ROI claims beyond the public docs.

If a reader needs private scoping, credentials, contract terms, payment, or custom implementation details, stop at a public-safe acknowledgment and get fresh approval before moving to any private channel or transaction step.
